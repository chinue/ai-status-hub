# 换行风格

## 默认规则（源代码和一般文本）

- 当**添加新文件或对非脚本文件进行大幅重写**时，**优先使用 Windows 风格换行**：`CRLF` (`\r\n`)。
- 如果项目或当前对话明确要求统一使用 LF，则遵循项目/用户的指示。

## 脚本文件（保留已有风格）

对于以下被视为"脚本"的文件类型，**不要强制转换为 CRLF**：编辑前后，换行风格应与文件中**已占主导的换行风格保持一致**（如果整个文件主要是 LF，则继续使用 LF；如果主要是 CRLF，则继续使用 CRLF）。

- **Shell / 类 Unix 脚本**：`**/*.sh`、`"**/*.bash`、`"**/*.zsh`、`"**/*.fish` 等。
- **Windows 脚本**：`**/*.ps1`、`"**/*.psm1`、`"**/*.bat`、`"**/*.cmd`。
- **无固定扩展名但通常视为脚本的文件**（如果仓库中存在）：例如 `configure`、`Makefile`、`Dockerfile` 等 —— **遵循该文件已有的换行风格**。
- **Shebang 脚本**：首行形如 `#!/usr/bin/env bash` 等 —— **遵循该文件已有的换行风格**。

## 如何判断"原始风格"

- 如果文件已包含 `\r\n`，且占多数或混合内容中 CRLF 占主导 → 继续使用 **CRLF**。
- 如果仅存在 `\n`（典型的 Linux/macOS 脚本）→ 继续使用 **LF**。
- **新建**脚本文件：在 Windows 仓库中可默认使用 **CRLF**（与默认规则一致）；如果脚本明确仅用于 Unix（例如只有 `*.sh` 且项目约定为 LF），则遵循项目约定。

## 与编辑器设置的关系

- 本规则是关于 **Agent 修改差异**的约定；本地的 `files.eol` 仍应遵循团队约定。当与本规则冲突时，**用户/项目文档**优先。
