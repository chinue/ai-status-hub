# 中文优先沟通规范

## 适用范围

本规则适用于以下所有场景：

- **Plan 文件**：通过 `EnterPlanMode` / `ExitPlanMode` 制定的任何实现计划、设计文档、技术方案。
- **用户回答**：直接向用户作出的所有文字回复。
- **思考与推理**：内部思考、分析、推理过程（即使是对话中的中间步骤）。

## 规则

1. **Plan 必须使用中文书写**
   - 计划标题、步骤描述、选项说明、权衡分析等全部使用中文。
   - 技术术语（如 API 名称、文件名、代码片段）可保留英文，但周围解释性文字必须为中文。

2. **回答问题必须使用中文**
   - 无论用户用何种语言提问，回答均使用中文。
   - 如果用户明确要求使用其他语言，则遵循用户指示。

3. **思考过程使用中文**
   - Agent 的内部推理、分析、总结应优先使用中文进行。
   - 这有助于保持与项目文档和用户语境的一致性。

## 例外情况

- 代码片段、命令行指令、配置文件内容保持其原始语言（通常为英文）。
- 引用外部英文文档时，引用内容可保留原文，但需附带中文解释。
- 用户明确指示使用其他语言时，遵循用户指示。
