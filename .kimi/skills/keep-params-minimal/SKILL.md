# 函数参数最小化规则（C/C++/CUDA）

## 原则

- **保持参数列表尽可能最小**：让函数签名仅表达调用者必须提供的最少信息，降低调用和维护成本。
- **如果某个值可以从已有参数或可访问的上下文中以低成本推导出来，就不要为其添加新参数**。

## 何时不应添加参数

- 不要为了传递可以从以下来源获取的 **bool / 标志 / 重复信息** 而添加新参数：  - 已有参数（例如 `pixfmt`、`stride`、`width` / `height`）通过直接推导获得；  - 调用链中已有结构体的字段（例如 `p->pixfmt`、`p->width`）以低成本获取；  - 简单的内部约定编码（例如在已有枚举或位域中添加内部标志）。

## 何时可以添加参数（例外情况）

- 推导需要**昂贵的计算、额外的依赖或明显的复杂性**；
- 推导会引入**歧义**或**隐藏的副作用**（例如必须访问全局状态、额外的 IO）；
- 该参数是**关键业务输入**，为可读性 / 可测试性应保持显式。

## 示例（源代码注释应使用英文）

```cpp
// BAD: 当 rowMul 可以被编码或推导时仍添加该参数
// _Create(width, height, pixfmt, rowMul);

// GOOD: 不添加额外参数；将 UV 处理意图编码到 pixfmt 中或从上下文推导
// _Create(width, height, pixfmt);
// CreateSemiUV(...) 在 pixfmt 上按位或一个内部标志，用于 semi-planar UV 处理。
```
