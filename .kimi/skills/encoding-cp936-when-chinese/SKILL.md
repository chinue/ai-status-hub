# 编码保存（含中文时使用 CP936/GBK）

当你在匹配的文件中**看到任何中文内容**（中文字符串、中文注释、中文 UI 文本等）并需要修改时：

- **必须**：保持/保存为 **CP936/GBK (Windows "ANSI")** 编码。
- **禁止**：将文件保存为 **UTF-8**（无论是否带 BOM），以避免乱码和 `warning C4819`。

## 具体要求（当 Agent 进行修改时）

- **编辑前检查编码**：如果文件已经是 CP936/GBK，修改后必须保持该编码不变。
- **不要引入非 CP936 字符**：尤其是 Unicode 标点符号（如全角破折号、弯引号）和不常用符号。
- **如果必须写入无法在 CP936 中表示的字符**：不要直接写入磁盘，而应使用 ASCII 近似字符或 `\uXXXX`/转义方案（根据语言/文件格式允许的方式）。

## 提示

- 当出现 C4819 时，首先检查文件是否被误存为 UTF-8，或者是否引入了 CP936 不支持的 Unicode 字符。
